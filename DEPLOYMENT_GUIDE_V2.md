# SRT语音合成工具 - v2版本（分段生成 + 音频拼接）

## 🆕 v2版本特性

### 🔧 核心改进
- **分段音频生成**: 每段字幕独立生成音频
- **精确时间轴对齐**: 根据时间间隔添加精确静音
- **音频拼接**: 使用pydub拼接所有音频段

### 🎯 优势
- ✅ **时间轴精确对齐**: 保持原始字幕的时间节奏
- ✅ **自然语音停顿**: 真实的音频间隔
- ✅ **高质量输出**: 避免文本拼接带来的语音不自然

### ⚠️ 注意事项
- ⚠️ **内存占用较大**: 需要同时处理多个音频文件
- ⚠️ **处理时间较长**: 分段生成需要更多时间
- ⚠️ **依赖ffmpeg**: 需要安装ffmpeg系统依赖

## 🚀 部署步骤

### 1. 构建v2版本镜像
```bash
# 构建v2版本
docker build -f Dockerfile.v2 -t srt-tts:v2 .
```

### 2. 使用Docker Compose部署
```bash
# 启动v2版本服务
docker-compose up srt-tts-v2 -d

# 查看服务状态
docker-compose ps
```

### 3. 访问服务
- **v1版本**: http://localhost:5000
- **v2版本**: http://localhost:5001

## 🔧 故障排除

### ffmpeg依赖问题
```bash
# 检查ffmpeg是否安装
ffmpeg -version

# 安装ffmpeg（Ubuntu/Debian）
sudo apt-get update
sudo apt-get install ffmpeg
```

### 内存不足
- 减少并发转换任务
- 增加系统内存
- 使用v1版本处理大文件

### 音频拼接失败
- 检查临时文件权限
- 验证pydub安装
- 查看服务日志

## 📈 使用建议

### 推荐使用v2版本的场景
- ✅ 需要精确时间轴对齐
- ✅ 对语音质量要求高
- ✅ 字幕数量较少（<50段）
- ✅ 有足够的内存资源

### 推荐使用v1版本的场景
- ✅ 处理大量字幕文件
- ✅ 内存资源有限
- ✅ 对处理速度要求高
- ✅ 简单的语音合成需求

## 🔄 版本切换

### 从v1切换到v2
```bash
# 停止v1服务
docker-compose stop srt-tts

# 启动v2服务
docker-compose up srt-tts-v2 -d
```

### 从v2切换回v1
```bash
# 停止v2服务
docker-compose stop srt-tts-v2

# 启动v1服务
docker-compose up srt-tts -d
```
