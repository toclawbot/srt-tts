# SRT语音合成工具 - JSON解析错误修复部署指南

## 问题描述
当上传的SRT文件过大（字幕过多）时，前端JavaScript会出现JSON解析错误：
```
JSON.parse: unexpected character at line 1 column 1 of the JSON data
```

## 修复内容

### 🔧 前端修复 (templates/index.html)
- ✅ 添加安全的JSON解析函数 `safeJsonParse()`
- ✅ 改进错误处理，提供更友好的错误信息
- ✅ 增加详细的错误日志记录

### 🔧 后端修复 (app.py)  
- ✅ 添加文件大小限制检查（50MB）
- ✅ 添加字幕行数限制检查（10000行）
- ✅ 添加字幕段数限制检查（1000段）
- ✅ 改进错误处理和文件清理

## 🚀 部署步骤

### 1. 更新代码
```bash
# 备份原文件
cp templates/index.html templates/index.html.backup
cp app.py app.py.backup

# 应用修复
# 前端: 已更新 templates/index.html 中的 pollProgress 函数
# 后端: 已更新 app.py 中的 convert() 函数
```

### 2. 测试修复
```bash
# 运行测试脚本
python3 test_fix.py
```

### 3. 重启服务
```bash
# 如果使用Docker
sudo docker-compose down
sudo docker-compose up -d

# 如果直接运行Python
python3 app.py
```

### 4. 验证修复
1. 上传一个正常大小的SRT文件 - 应该正常工作
2. 上传一个超过50MB的大文件 - 应该显示友好的错误信息
3. 上传一个有大量字幕的文件 - 应该显示字幕数量限制错误

## 📊 限制说明

| 限制项 | 最大值 | 说明 |
|--------|--------|------|
| 文件大小 | 50MB | 防止内存溢出和服务器过载 |
| 字幕行数 | 10,000行 | 防止解析器性能问题 |
| 字幕段数 | 1,000段 | 防止语音合成时间过长 |
